# TypeWorks — 타자연습 웹사이트

한국어(2벌식) · 영어 · 일본어(로마자 입력) 타자연습을 지원하는 정적 웹사이트입니다.
서버 없이 `index.html`을 브라우저에서 바로 열기만 하면 동작합니다.

## 빠른 시작

`index.html`을 더블클릭해서 브라우저로 열면 바로 사용할 수 있습니다.
(별도 설치나 서버 구동이 필요 없습니다.)

## 폴더 구조

```
typing-practice/
├── index.html
├── css/style.css
├── js/
│   ├── hangul.js         # 한글 자모 분해·비교 로직
│   ├── romaji.js         # 일본어 로마자 입력 매칭 로직
│   ├── data-manifest.js  # /data 스캔 결과 (자동 생성 파일, 직접 수정 금지)
│   └── app.js             # 화면 라우팅, 타이핑 엔진, 결산화면, 설정
├── data/
│   ├── korean/{작가명}/{제목}.txt
│   ├── english/{작가명}/{제목}.txt
│   └── japanese/{작가명}/{제목}.txt
└── scripts/build-manifest.js  # /data → js/data-manifest.js 생성 스크립트
```

## 지문 추가하는 방법

1. `data/korean`, `data/english`, `data/japanese` 폴더 아래에 원하는 작가명 폴더를 만들고, 그 안에 `.txt` 파일을 넣습니다.
   - 파일명이 그대로 "제목"이 됩니다.
   - 파일 내용이 본문(지문)이 됩니다.
2. **일본어**는 형식이 조금 다릅니다. 파일 안에 원문과 후리가나(전체 히라가나 읽기)를 `---` 구분선으로 나눠 적어야 합니다.

   ```
   朝、窓を開けると、冷たい風が入ってきた。
   ---
   あさ、まどをあけると、つめたいかぜがはいってきた。
   ```

   - 원문 줄의 문장 개수(。！？ 기준)와 읽기 줄의 문장 개수가 반드시 같아야 "무작위 짧은글" 풀에 포함됩니다.
   - `---` 구분선이 없으면 원문 자체가 읽기로 취급됩니다(순수 가나 문장일 때만 사용하세요).

3. 파일을 추가/삭제/수정한 뒤에는 아래 명령을 실행해 목록을 다시 생성해야 사이트에 반영됩니다.

   ```
   node scripts/build-manifest.js
   ```

   (Node.js가 필요합니다. 없다면 https://nodejs.org 에서 설치하세요.)

## 기능 요약

- **긴 글 연습**: 메뉴에서 작가 → 제목을 선택해 글 전체를 처음부터 끝까지 입력합니다.
- **무작위 짧은글 연습**: 모든 지문에서 문장 단위로 무작위 발췌해 빠르게 연습합니다.
- **한국어**: 2벌식 기준, 자모(초성/중성/종성) 단위로 정오를 판정합니다.
- **일본어**: 원문(한자 포함)과 후리가나(읽기)를 함께 보여주고, 로마자로 입력합니다.
  헵번식을 기본으로 하되 `shi/si`, `chi/ti`, `tsu/tu`, `fu/hu`, `ji/zi` 등 흔한 표기 변형과
  촉음(っ→중복자음), 발음(ん→n/nn/n'), 장음(ー) 표기를 폭넓게 허용합니다.
  다만 모든 IME 변형(예: 삼중 n, `wo` 대신 `o` 등 극히 드문 표기)까지 완전히 다 지원하지는 않습니다.
- **결산 화면**: 최종 CPM/WPM, 평균 타수, 오타율, 소요시간과 함께 시간대별 타수 변화 그래프, 최근 기록을 보여줍니다.
- **설정**: 라이트/다크 테마, 문장부호 자동 보정(On 시 문장 끝 마침표 등이 생략되어도 정타로 인정) — 모두 브라우저 로컬 저장소에 저장되어 재접속 시에도 유지됩니다.

## 알아두면 좋은 점

- 문단 구분을 위한 줄바꿈은 연습 중에는 공백으로 합쳐져 한 줄로 표시됩니다(입력창이 한 줄 입력 방식이기 때문입니다).
- 기록/설정은 브라우저의 `localStorage`에 저장되므로, 브라우저나 기기를 바꾸면 기록이 유지되지 않습니다.
- 실제 서비스로 배포하려면 이 폴더 전체를 정적 호스팅(GitHub Pages, Netlify, Vercel 등)에 올리기만 하면 됩니다.
